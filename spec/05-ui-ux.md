## 5.2 Screen states / flow
- **Empty/idle** (drop-or-browse invitation, local/offline reassurance, no
  setup) → **collected/confirm** (detected format + count) → **targets+options**
  (pre-highlighted default, Advanced options) → **destination** ("will save to…")
  → **re-run prompt** (when §2.5 flags equivalent output exists — one batch-level
  interstitial, skip-default / fresh-copy) → **progress** (per-item + batch,
  cancellable) → **summary** (success/fail, open folder/file). Plus **mixed-drop
  refusal** and **unsupported/unreadable** states. State machine diagram.
- A **patent-gapped/unavailable target** (per the §3.4 matrix) appears in the
  picker consistently — decide omitted vs disabled-with-note — sourced from §3.4.
  _(expand)_
